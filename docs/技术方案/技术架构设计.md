# Chamate 技术架构设计

## 1. 系统架构概览

Chamate采用**前后端分离**架构，整体数据流：

```
┌─────────────────┐     HTTP/SSE      ┌─────────────────┐
│    前端应用     │ ←───────────────→ │    后端服务      │
│   (Next.js)     │                   │   (FastAPI)      │
└────────┬────────┘                   └────────┬────────┘
         │                                     │
         │ LocalStorage                        │
         │ (加密API Key)                       ├────→ SQLite (数据持久化)
         │                                     │
         └─────────────────────────────────────┼────→ OpenAI API
                                               ├────→ Claude API
                                               ├────→ 通义千问 API
                                               └────→ 智谱AI API
```

**架构要点**：
- 前端负责UI交互、API Key加密存储
- 后端负责业务逻辑、AI调用、数据持久化
- 前端通过HTTP Header传递加密API Key，后端解密后调用AI服务
- SSE实现流式对话响应

## 2. 模块划分

### 2.1 前端模块

| 模块 | 职责 | 关键内容 |
|------|------|----------|
| **页面路由** | App Router页面组织 | `/chat`, `/debate`, `/settings` |
| **UI组件** | 可复用界面组件 | ChatPanel, MessageBubble, DebateView, ModelSelector |
| **状态管理** | 客户端状态 | Zustand管理会话列表、当前对话、设置项 |
| **工具库** | 通用功能 | 加密工具、SSE客户端、API封装 |

### 2.2 后端模块

| 模块 | 职责 | 关键内容 |
|------|------|----------|
| **API层** | 路由与请求处理 | FastAPI Router, 请求校验, 响应封装 |
| **服务层** | 业务逻辑 | ConversationService, DebateService, ChatService |
| **Provider层** | AI接口适配 | BaseProvider抽象 + 各厂商实现 |
| **数据层** | 数据持久化 | SQLAlchemy Models, Repository |

**Provider层设计理念**：
- 定义统一接口：`stream_chat(messages, model, api_key)`
- 各Provider实现适配各厂商API差异
- 上层服务无感知底层API差异

|------|------|------|
| id | UUID | 主键 |

| 字段 | 类型 | 说明 |
|------|------|------|

| topic | String | 辩论主题 |
| user_stance | Enum | pro / con / neutral |
| status | Enum | pending / in_progress / completed |
| created_at | DateTime | 创建时间 |
| updated_at | DateTime | 更新时间 |

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |

| con_argument | Text | 反方论点 |

## 4. API接口设计

## 5. 安全设计

### 5.1 API Key存储方案

```
┌─────────────┐    加密     ┌─────────────────────┐
│  用户输入   │ ─────────→ │  LocalStorage存储    │
│  API Key    │  AES-256   │  (密文)              │
└─────────────┘            └──────────┬──────────┘
                                      │
                                      │ 请求时
                                      ↓
                           ┌─────────────────────┐
                           │  X-API-Key Header   │
                           │  (密文传递)          │
                           └──────────┬──────────┘
                                      │
                                      ↓
                           ┌─────────────────────┐
                           │  后端解密使用        │
                           │  (内存中临时存在)    │
                           └─────────────────────┘
```

**加密方案要点**：
- 算法：AES-256-GCM
- 密钥派生：基于设备指纹 + 用户密码（可选）
- 后端不持久化API Key，仅运行时解密使用

### 5.2 请求安全

| 措施 | 说明 |
|------|------|
| HTTPS | 传输层加密 |
| CORS | 限制跨域来源 |
| Rate Limit | 请求频率限制 |
| Input Validation | Pydantic模型校验 |

## 6. 部署架构

### Docker Compose编排

```yaml
services:
  frontend:
    image: chamate-frontend
    ports: ["3000:3000"]
    depends_on: [backend]

  backend:
    image: chamate-backend
    ports: ["8000:8000"]
    volumes:
      - ./data:/app/data  # SQLite持久化
    environment:
      - DATABASE_URL=sqlite:///./data/chamate.db
```

**部署特点**：
- 单机Docker Compose一键启动
- 前端静态资源可选CDN加速
- 数据卷挂载确保SQLite数据持久化
- 支持环境变量配置切换数据库

### 资源需求（MVP）

| 组件 | CPU | 内存 |
|------|-----|------|
| Frontend | 0.5核 | 512MB |
| Backend | 1核 | 1GB |
| 总计 | 1.5核 | 1.5GB |
