## 一、产品简介

### 1.1 产品概述

本产品是一款**本地自托管的AI智能对话平台**，为用户提供安全、私密、可定制的AI对话体验。产品支持主流云端AI模型接入，具备辩论/头脑风暴等高级交互模式，并可通过MCP协议扩展工具能力。

### 1.2 产品愿景

> 让每个人都能拥有安全、智能、可深度协作的AI思考伙伴

### 1.3 核心价值主张

| 价值维度           | 说明                                             |
| ------------------ | ------------------------------------------------ |
| **隐私安全** | 数据本地存储，API Key加密管理，完全掌控个人数据  |
| **深度交互** | 支持辩论、头脑风暴等高级交互模式，超越简单问答   |
| **灵活定制** | 多模型切换、工具扩展、技能管理，满足个性化需求   |
| **记忆延续** | 分层记忆系统，跨会话上下文延续，打造个性化AI伙伴 |

## 三、功能模块详解

### 3.1 基础对话功能

#### 3.1.1 多模型切换

**功能描述：** 用户可在对话中自由切换不同的AI模型提供商和具体模型。

**支持模型：**

| 提供商    | 支持模型                     | 说明               |
| --------- | ---------------------------- | ------------------ |
| OpenAI    | GPT-4o、GPT-4、GPT-3.5-turbo | 通用能力强         |
| Anthropic | Claude-4、Claude-3.5-Sonnet  | 长文本、代码能力强 |
| 通义千问  | Qwen-Max、Qwen-Plus          | 中文场景优化       |
| 智谱AI    | GLM-4、GLM-Edge              | 性价比高           |
| Ollama    | 用户自选本地模型             | 预留接口，按需启用 |

**用户操作：**

1. 在对话界面顶部选择模型提供商
2. 选择具体模型版本
3. 切换后新消息使用新模型，历史对话保持不变

#### 3.1.2 API Key管理

**功能描述：** 用户可在前端自行配置和管理各模型的API Key，数据本地加密存储。

**功能特性：**

- 🔐 **加密存储**：API Key使用加密算法存储于浏览器IndexedDB
- 🔄 **随时修改**：用户可随时更新或删除已配置的API Key
- ✅ **连接测试**：配置后可测试API连接是否成功
- 📋 **多Key管理**：支持同一提供商配置多个Key（如不同账户）

**配置界面：**

```

│                      API Key 配置中心                            │

│  通义千问                                                       │
```

#### 3.1.3 对话历史管理

**功能描述：** 自动保存所有对话历史，支持搜索、分类、删除等操作。

**功能特性：**

- 📁 **会话列表**：左侧边栏显示所有历史会话
- 🔍 **搜索功能**：支持按关键词搜索历史对话内容
- 🏷️ **标签分类**：用户可给会话添加标签便于分类
- 🗑️ **批量删除**：支持选择多个会话批量删除
- 📤 **导出功能**：支持导出对话历史为Markdown/PDF格式

### 3.2 高级交互功能 ⭐核心特色

#### 3.2.1 辩论模式

**功能描述：** 用户提出观点后，AI自动扮演正反双方进行多轮辩论，帮助用户完善观点、发现盲点。

**适用场景：**

- 观点验证与完善
- 决策方案评估
- 学术论点推敲
- 商业策略分析

**模式类型：**

| 模式     | 说明                 | 适用场景     |
| -------- | -------------------- | ------------ |
| 单AI辩论 | 同一AI扮演正反双方   | 快速观点检验 |
| 多AI辩论 | 不同模型代表不同立场 | 深度观点碰撞 |
| 人机协作 | 用户+AI vs AI        | 用户参与辩论 |

### 3.4 工具与扩展功能

#### 3.4.1 MCP工具调用

**功能描述：** 通过MCP（Model Context Protocol）协议，AI可调用各种外部工具完成任务。

**内置工具：**

| 工具     | 功能                   | 使用场景               |
| -------- | ---------------------- | ---------------------- |
| 联网搜索 | 搜索互联网获取最新信息 | 查询新闻、资料检索     |
| 文件读写 | 读取和写入本地文件     | 文档处理、数据导入导出 |
| 代码执行 | 在沙箱中执行代码       | 数据分析、自动化脚本   |
| 网页浏览 | 访问和解析网页内容     | 信息抓取、内容分析     |

**工具使用流程：**

1. 用户在设置中启用所需工具
2. AI在对话中自动判断是否需要调用工具
3. 调用前向用户确认（可配置为自动确认）
4. 执行工具并返回结果
5. AI基于工具结果继续对话

#### 3.4.4 RAG检索增强

**功能描述：** 用户可上传文档资料，AI在回答时自动检索相关内容，提供基于用户知识的增强回答。

**支持文档格式：**

- 📄 PDF文档
- 📝 Markdown文件
- 📊 Word文档
- 🌐 网页链接

**RAG使用流程：**

1. 用户上传文档或添加知识库
2. 系统自动分块、向量化存储
3. 用户提问时，系统检索相关文档片段
4. AI基于检索内容生成增强回答
5. 回答中标注引用来源

## 五、数据与隐私

### 5.1 数据存储说明

| 数据类型     | 存储位置             | 加密方式    | 用户控制            |
| ------------ | -------------------- | ----------- | ------------------- |
| API Key      | 本地浏览器IndexedDB  | AES-256加密 | 用户可随时修改/删除 |
| 对话历史     | 本地PostgreSQL数据库 | 可选加密    | 用户可导出/删除     |

| 用户上传文档 | 本地文件系统         | 可选加密    | 用户可管理          |

## 八、常见问题

### 8.1 FAQ

| 问题                    | 答案                                                            |
| ----------------------- | --------------------------------------------------------------- |
| 需要自己准备API Key吗？ | 是的，用户需要自行准备各模型的API Key，费用直接支付给模型提供商 |
| 数据会上传到云端吗？    | 不会，所有数据（除API调用外）均存储于本地                       |
| 可以离线使用吗？        | 基础界面可离线访问，但对话功能需要网络访问云端API               |
| 支持哪些模型？          | OpenAI、Claude、通义千问、智谱AI等主流云端模型，Ollama预留      |
| 如何备份数据？          | 提供数据导出功能，可备份对话历史和配置                          |
| 可以多人使用吗？        | 当前版本为单用户设计，多用户支持在规划中                        |
